# Система контроля и управления доступом (СКУД)

Система контроля и управления доступом с контекстно-зависимой динамической аутентификацией на основе HMAC-SHA256 и ранговой системы.

## 📋 Описание

Данный проект представляет собой прототип системы СКУД, реализующий оригинальный метод аутентификации на основе:
- **Ранговой комбинации** (покерные комбинации от 3 до 9)
- **Истории перемещений** (контекстная проверка маршрута)
- **HMAC-SHA256 токена** с криптографически стойким nonce
- **Защиты от replay-атак** и brute-force

## 🔑 Ключевые особенности

- ✅ Аутентификация на основе HMAC-SHA256
- ✅ Ранговая система (3-9) на основе покерных комбинаций
- ✅ Контекстно-зависимый контроль доступа
- ✅ История перемещений пользователей
- ✅ Защита от replay-атак (nonce)
- ✅ Блокировка после 3 неудачных попыток (1 минута)
- ✅ Веб-интерфейс для управления и мониторинга
- ✅ CSRF-защита форм
- ✅ Логирование всех событий
- ✅ **Управление пользователями** (добавление, редактирование, удаление)
- ✅ **Выбор пользователя из списка** в эмуляторе прохода
- ✅ **Интерактивная схема офиса** с реальным временем
- ✅ **Отображение местоположения** всех пользователей по зонам
- ✅ **REST API** для интеграции с другими системами
- ✅ **NFC-интеграция** с PN532 для Raspberry Pi 🆕
- ✅ **Фоновый NFC-сервис** с автозапуском
- ✅ **Real-time опрос** NFC-считывателя

```bash

sudo ./install_systemd.sh
```

## 📖 Использование

### 1. Персонализация пользователя

Добавление нового пользователя в систему:

```bash
python personalize.py --uid ADMIN_01 --rank 9 --name "Администратор"
```

**Параметры:**
- `--uid` — уникальный идентификатор (например, ID карты NFC)
- `--rank` — ранг пользователя (3-9):
  - 3: Посетитель
  - 4-5: Основной персонал
  - 6-7: Технический персонал / менеджеры
  - 8-9: Администраторы / ИБ
- `--name` — имя для логирования (необязательно)

### 2. Запуск веб-интерфейса

```bash
python app.py
```

Откройте в браузере: **http://localhost:5000**

**Возможности веб-интерфейса:**
- 🚪 Эмуляция прохода через СКУД
- 👥 Просмотр пользователей
- 📋 Журнал событий доступа
- 🗺️ Схема офиса

### 3. Запуск тестов

```bash
python test_full.py
```

Тесты проверяют:
- ✅ Вычисление покерных рангов
- ✅ Аутентификацию пользователей
- ✅ Проверку недостаточного ранга
- ✅ Контекстную проверку (вход после выхода)
- ✅ Валидацию истории перемещений
- ✅ Проверку маршрутов
- ✅ Блокировку после неудачных попыток

## 🏗️ Архитектура

### Модель привилегий

| Ранг | Название | Доступ |
|------|----------|--------|
| 3 | Посетитель | Только вход/выход |
| 4-5 | Персонал | Офисные помещения |
| 6-7 | Техник/Менеджер | Склад, офисы |
| 8-9 | Администратор | Все зоны, включая серверную |

### Зоны по умолчанию

| ID | Название | Требуемый ранг |
|----|----------|----------------|
| 0 | Вход | ≥3 |
| 1 | Офис | ≥5 |
| 2 | Серверная | ≥8 |
| 999 | Выход | ≥3 |

### Процесс аутентификации

1. Считыватель генерирует криптографический nonce
2. Вычисляется токен: `T = HMAC-SHA256(K, nonce || history)`
3. Из токена извлекается комбинация из 5 чисел
4. Вычисляется ранг комбинации (покерные правила)
5. Доступ разрешён, если:
   - Ранг совпадает с рангом пользователя
   - Ранг ≥ требуемого ранга зоны
   - История перемещений корректна
   - Контекст проверки пройден

### Защита от атак

| Тип атаки | Мера защиты |
|-----------|-------------|
| Replay-атака | Одноразовый nonce |
| Brute-force | Блокировка после 3 попыток |
| CSRF | Токены в формах |
| SQL injection | JSON-сериализация, параметризованные запросы |
| Гонка условий | Транзакции с блокировкой |

## 🗄️ База данных

### Таблицы

- **users** — пользователи (UID, ранг, секретный ключ, текущая зона)
- **zones** — зоны доступа (ID, название, требования ранга)
- **access_logs** — журнал всех событий доступа
- **access_blocks** — временные блокировки пользователей
- **pending_passes** — активные сессии прохода (10 секунд)
- **user_history** — история перемещений пользователей

### Индексы

Созданы индексы для ускорения выборки:
- `idx_access_logs_uid`
- `idx_access_logs_timestamp`
- `idx_access_logs_uid_timestamp`
- `idx_user_history_uid`
- `idx_user_history_timestamp`
- `idx_pending_passes_authorized_at`

## 🧪 Тестирование

### Сценарии для проверки

1. ✅ Успешный проход с достаточным рангом
2. ✅ Отказ из-за недостаточного ранга
3. ✅ Отказ из-за нарушения контекста (вход после выхода)
4. ✅ Блокировка после 3 неудачных попыток
5. ✅ Отказ для несуществующего UID
6. ✅ Отказ для несуществующей зоны
7. ✅ Отказ при повреждённой истории

## 🌐 REST API

Система предоставляет следующие API endpoints:

### `GET /api/users`
Получить список всех пользователей с их текущими зонами.

**Ответ:**
```json
{
  "users": [
    {
      "uid": "ADMIN1",
      "rank": 9,
      "current_zone": 0,
      "zone_name": "Вход",
      "created_at": "2026-04-01 09:26:56"
    }
  ]
}
```

### `GET /api/zones`
Получить информацию обо всех зонах.

**Ответ:**
```json
{
  "zones": [
    {
      "id": 0,
      "name": "Вход",
      "is_exit": false,
      "required_rank": 3
    }
  ]
}
```

### `GET /api/status`
Получить статус системы — пользователи по зонам (используется для real-time обновления).

**Ответ:**
```json
{
  "zones": {...},
  "users_by_zone": {
    "0": [{"uid": "ADMIN1", "rank": 9}],
    "1": []
  },
  "total_users": 1
}
```

### 🆕 NFC API

#### `GET /api/nfc/status`
Проверить доступность NFC-ридера.

**Ответ:**
```json
{
  "available": true,
  "initialized": true
}
```

#### `POST /api/nfc/read`
Считать UID карты (однократно).

**Параметры:**
```json
{
  "timeout": 1000  // таймаут в мс
}
```

#### `POST /api/nfc/poll`
Опросить NFC-ридер и обработать доступ.

**Параметры:**
```json
{
  "zone_to": 1  // зона доступа
}
```

**Ответ (успех):**
```json
{
  "success": true,
  "action": "granted",
  "uid": "04A1B2C3",
  "message": "Доступ разрешён в зону \"Офис\"",
  "zone_from": 0,
  "zone_to": 1,
  "zone_name": "Офис"
}
```

**Ответ (отказ):**
```json
{
  "success": false,
  "action": "denied",
  "uid": "04A1B2C3",
  "message": "Недостаточно прав: ранг 4 < 5"
}
```

## 🎨 Новые функции веб-интерфейса

### Управление пользователями
- **Добавление**: через веб-форму на странице `/users`
- **Редактирование**: кнопка "✏️ Редактировать" рядом с пользователем
- **Удаление**: кнопка "🗑️ Удалить" с подтверждением

### Эмулятор прохода
- **Выпадающий список**: выбор существующего пользователя из списка
- **Ручной ввод**: возможность ввести UID вручную
- **Статус системы**: панель с текущим распределением пользователей по зонам

### Интерактивная схема офиса (`/dashboard`)
- **Визуальное отображение**: план офиса с помещениями
- **Real-time обновление**: каждые 3 секунды
- **Пользователи на карте**: кружки с рангом в каждой зоне
- **Подсказки**: при наведении на пользователя показывается информация
- **Статистика**: количество пользователей и занятых зон
- **Список по зонам**: подробный список всех пользователей в каждой зоне
